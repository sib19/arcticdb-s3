## 📋 The Three GitHub Methods

All documented in the guides:

### Method 1: HTTPS (Easiest)
```bash
git remote add origin https://github.com/YOUR_USERNAME/arcticdb-project.git
git branch -M main
git push -u origin main
```
**Best for:** Quick setup, first-timers
**Password:** Personal Access Token (not GitHub password)

### Method 2: SSH (Most Secure)
```bash
git remote add origin git@github.com:YOUR_USERNAME/arcticdb-project.git
git branch -M main
git push -u origin main
```
**Best for:** Frequent pushes, security-conscious
**Password:** None (one-time setup)

### Method 3: GitHub CLI (Fastest)
```bash
cd arcticdb-project
gh repo create arcticdb-project --source=. --remote=origin --push
```
**Best for:** One-command setup
**Password:** GitHub login

## 🛠️ Common Commands Cheat Sheet

### Git Commands
```bash
# Setup (one time)
git remote add origin <URL>
git branch -M main
git push -u origin main

# Daily work
git status                    # Check what changed
git add .                     # Stage changes
git commit -m "message"       # Commit changes
git push                      # Push to GitHub

# Checking
git log                       # View history
git remote -v               # View remotes
git branch                  # View branches
```

### Docker Commands
```bash
# Start
docker-compose up -d        # Start container
docker-compose logs -f      # View logs
docker-compose ps          # Check status

# Stop
docker-compose down         # Stop container

# Rebuild
docker-compose up -d --build  # Rebuild image
```

### API Commands
```bash
# Health check
curl http://localhost:8080/health

# Write data
curl -X POST http://localhost:8080/api/v1/write ...

# Read data
curl http://localhost:8080/api/v1/read/table_name

# S3 upload
curl -X POST http://localhost:8080/api/v1/s3/upload ...
```

### Make Commands
```bash
make start          # Start Docker
make stop           # Stop Docker
make logs           # View logs
make examples       # Run examples
make health         # Check health
make test           # Run tests
```

## 🔒 Security Reminders

✅ DO:
- Use Personal Access Token for HTTPS (not password)
- Keep .env file locally (in .gitignore)
- Use SSH keys when possible
- Rotate credentials regularly
- Use IAM roles for AWS

❌ DON'T:
- Commit .env file with secrets
- Share Personal Access Token
- Use GitHub password for git
- Push private keys to GitHub
- Store credentials in code
